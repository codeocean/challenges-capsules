# Claude Code Mission: Challenge 02 - Agentic Data Harmonization

## Mission
Replace the existing deterministic fuzzy-matching pipeline with a genuinely agentic data harmonization system using Strands Agents SDK + Bedrock Claude. The current code FAILS every audit criterion: no LLM calls, circular evaluation, 72% scope exclusion, no entity resolution.

## Current State
Read ALL existing code first. The capsule has run.py with hardcoded override maps and rapidfuzz matching. DELETE the existing run.py and rebuild.

## Required Changes (in order)

### 1. New Modular Project Structure
Create under /code/:
- run.py (orchestrator)
- agent/ (agent_factory.py, mapping_agent.py, schema_agent.py)
- harmonization/ (schema_profiler.py, schema_reconciler.py, fuzzy_matcher.py, entity_resolver.py, data_quality.py, merger.py)
- evaluation/ (gold_standard.py, metrics.py)
- provenance/ (logger.py)
- config.py

### 2. Gold Standard (100+ entries)
Create an independent gold standard CSV with:
- 60 easy mappings (exact/near-exact)
- 40+ medium/hard mappings (synonyms, semantic)
- 20+ "no match" entries
- NOT generated by the pipeline itself

### 3. Schema Profiling & Reconciliation
- Profile both input datasets
- Use Strands Agent to reconcile column-level schema differences
- Output schema_reconciliation.json with agent reasoning

### 4. Deterministic Pre-Matching
- Run rapidfuzz on ALL 4290 labels (zero scope exclusions)
- High confidence (>95) → auto-accept
- 70-95 → escalate to agent
- <70 → escalate with low-confidence flag

### 5. Agentic Mapping (3 rounds)
- Round 1: Initial mapping with ontology tools
- Round 2: Self-correction (consistency review of CL terms with 3+ labels)
- Round 3: Review queue for low-confidence

### 6. Entity Resolution
- At minimum: blocking on harmonized cell type + brain region
- Probabilistic matching or agent escalation for ambiguous cases
- Output entity_linkage.csv

### 7. Data Quality Enforcement
- Null check, type check, ontology validity, duplicate check, consistency check
- Output data_quality_report.json

### 8. Harmonized Output
- Merge both datasets into harmonized.parquet
- Include provenance columns: source_dataset, original_label, mapping_method, confidence

### 9. Evaluation
- Evaluate against gold standard
- Precision, recall, F1 must NOT be 1.0 (that means circular)
- Target: precision >= 0.70, recall >= 0.60

### 10. Provenance
- JSONL log covering all phases
- At least 100 agent entries with reasoning

## Constraints
- Use Strands Agents SDK (strands-agents) with BedrockModel only
- NO direct anthropic or openai imports
- NO hardcoded override dictionaries
- Process ALL labels (>= 4000 of 4290)
- All outputs to /results/
- Backup code to /results/backup/code/
